# Design: Simplify the menu-bar IME indicator icons (monochrome template)

Date: 2026-06-03
Status: Built & installed (final design below). Scope: macOS app target (`OSX`),
asset catalog only — no Info.plist, no source-code change.

## Goal

Make the menu-bar input-mode indicator icons simpler and more macOS-native by
converting them from colored, slightly skeuomorphic cloud+label icons into flat
**monochrome template** icons that auto-adapt to the light/dark menu bar.

## Final design (as built)

A flat, monochrome **cloud**, distinguished only by Korean vs English — **no
inner text**:

- **Korean (ko) modes → FILLED cloud** (SF Symbol `cloud.fill`).
- **English/roman (en) modes → OUTLINE cloud** (SF Symbol `cloud`).

Rationale for dropping the per-layout label: the active mode's name is already
shown beside the icon in the input menu and the keyboard/input-source picker, so
the always-visible menu bar only needs to convey Korean-vs-English at a glance.
Filled vs outline carries that distinction with zero text, which stays clean at
16 px. (This replaced an earlier "cloud + single-character knockout label"
concept that proved busy at 16 px and whose hand-drawn/clipped cloud shapes read
as a house or a block — see Design evolution.)

### Mode → cloud mapping (per Info.plist `TISIntendedLanguage`)

| cloud   | icon files                                  | input modes |
|---------|---------------------------------------------|-------------|
| OUTLINE (`cloud`)      | `eng`, `qwerty`               | system, colemak, dvorak, qwerty (all `en`) |
| FILLED (`cloud.fill`)  | `han`, `han2`, `han3`, `han390`, `han3final`, `hanroman` | every `ko` mode |

All Korean imagesets carry identical filled-cloud artwork; both English
imagesets carry identical outline-cloud artwork. Layout variants (두벌식 /
세벌식 / 390 / 최종 / 로마자) are NOT distinguished in the menu bar — only by name
in the menus. Accepted per user direction.

### Sizing

The cloud's tight opaque bounds are aspect-fit to the full tile (`margin = 0`,
no clipping). cloud.fill / cloud are wider than tall, so this fills the tile
width edge-to-edge at the cloud's natural proportion (~70% height). Going larger
requires clipping the sides, which cuts the outline stroke and distorts the
silhouette — rejected on device. This no-clip fit is the chosen size.

## Current state (what existed before)

- 8 distinct icon files, each an imageset in `OSX/Assets.xcassets`, referenced
  by filename from `OSX/Info.plist` (`tsInputModeMenuIconFileKey` etc.).
- The PNGs ship only inside the compiled `Assets.car` (no loose copies). The
  Text Input system resolves the names through the asset catalog, so replacing
  the imageset PNGs is sufficient — no Info.plist edit, no re-registration.
- Each imageset has `@1x` (16 px) + `@2x` (32 px), `universal` idiom.
- Old style: colored cloud (Korean blue, English orange) with a small label
  below or on the cloud.

## Dark-mode mechanism

Each PNG is rendered **black + alpha** (opaque black cloud / outline; transparent
elsewhere). Each imageset's `Contents.json` is marked:

```json
"properties" : { "template-rendering-intent" : "template" }
```

so AppKit tints the shape to the destination appearance (black on light bars,
white on dark bars). On-device verification in both appearances is the final
acceptance check; the fallback (explicit light/dark appearance variants in the
catalog) was not needed.

## Generation (reproducibility)

Icons are generated by a committed AppKit/Swift script,
`OSX/Icons/generate-menubar-icons.swift`:

- Renders `cloud.fill` (Korean) and `cloud` (English) to static PNGs at
  generation time — no runtime SF Symbol dependency.
- Computes each symbol's tight opaque bounds and aspect-fits them to the tile.
- Normalizes antialiased symbol pixels to opaque black (template mask).
- Writes all 16 PNGs (8 imagesets × @1x/@2x). Re-run:
  `swift OSX/Icons/generate-menubar-icons.swift`.

## Scope of change

Only the 8 menu-bar imagesets (`eng, han, han2, han3, han390, han3final,
hanroman, qwerty`), both scales, plus their `Contents.json` (template intent),
plus the generator script. Unchanged: `OSX/Info.plist`, all source,
`AppIcon.appiconset`, `OSXTestApp`, `Preferences/Example.png`, `OSX/Icons/*.png`.

## Design evolution (for the record)

Iterated on device with the user:
1. Cloud + single-char knockout label (2/3/9/F/R/Q/안), hand-drawn bezier cloud
   → read as a "house"; 16 px labels cramped; 안 illegible.
2. SF `cloud.fill` silhouette → cleaner cloud, but aspect-fit looked small;
   height-filling it clipped the sides into a block.
3. **Final:** drop labels entirely; distinguish Korean/English by filled vs
   outline cloud; size by no-clip aspect-fit.

## Risks / open items

- **R1: template tinting in the system IME menu** — verify white-on-dark on
  device. Fallback (explicit dark appearance variants) available if needed.
- Layout variants are intentionally indistinguishable in the menu bar (only by
  name in menus). Accepted.

## Success criteria

- All 8 menu-bar indicators are flat monochrome templates, correct in light and
  dark menu bars on device.
- Korean vs English is distinguishable at a glance (filled vs outline).
- No Info.plist or code change; icons regenerable from a committed script.
